pyTooling.Filesystem

An object-oriented file system abstraction for directory, file, symbolic link, … statistics collection.

Important

This isn’t a replacement of pathlib introduced with Python 3.4.

Exceptions

  • FilesystemException: Base-exception of all exceptions raised by pyTooling.Filesystem.

  • Base: Undocumented.

  • Element: Abstract base class for generic types.

  • Filename: Abstract base class for generic types.

  • SymbolicLink: Abstract base class for generic types.

class pyTooling.Filesystem.Directory[source]

A directory represents a directory in the filesystem contains subdirectories, regular files and symbolic links.

While scanning for subelements, the directory is populated with elements. Every file object added, gets registered in the filesystems Root for deduplication. In case a file identifier already exists, the found filename will reference the same file objects. In turn, the file objects has then references to multiple filenames (parents). This allows to detect hardlinks.

The time needed for scanning the directory and its subelements is provided via ScanDuration.

After scnaning the directory for subelements, certain directory properties get aggregated. The time needed for aggregation is provided via AggregateDuration.
