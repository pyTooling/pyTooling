pyTooling.Versioning

Implementation of semantic and date versioning version-numbers.

Hint

See high-level help for explanations and usage examples.

Functions

  • WordSizeValidator(): A factory function to return a validator for Version instances for a positive integer range based on word-sizes in bits.

  • MaxValueValidator(): A factory function to return a validator for Version instances checking for a positive integer range [0..max].

Classes

  • Parts: Enumeration describing parts of a version number that can be present.

  • ReleaseLevel: Enumeration describing the versions maturity level.

  • Flags: State enumeration, if a (tagged) version is build from a clean or dirty working directory.

  • Version: Base-class for a version representation.

  • SemanticVersion: Representation of a semantic version number like 3.7.12.

  • PythonVersion: Represents a Python version.

  • CalendarVersion: Representation of a calendar version number like 2021.10.

  • YearMonthVersion: Representation of a calendar version number made of year and month like 2021.10.

  • YearWeekVersion: Representation of a calendar version number made of year and week like 2021.47.

  • YearReleaseVersion: Representation of a calendar version number made of year and release per year like 2021.2.

  • YearMonthDayVersion: Representation of a calendar version number made of year, month and day like 2021.10.15.

  • RangeBoundHandling: A flag defining how to handle bounds in a range.

  • VersionRange: Representation of a version range described by a lower bound and upper bound version.

  • VersionSet: Representation of an ordered set of versions.
