pyTooling.Warning

A solution to send warnings like exceptions to a handler in the upper part of the call-stack.

Hint

See high-level help for explanations and usage examples.

Exceptions

  • UnhandledWarningException: Base exception derived from Exception for all custom exceptions.

  • UnhandledCriticalWarningException: Base exception derived from Exception for all custom exceptions.

  • UnhandledExceptionException: Base exception derived from Exception for all custom exceptions.

Classes

  • Warning: Common base class for all exceptions

  • CriticalWarning: Common base class for all exceptions

  • WarningCollector: A context manager to collect warnings within the call hierarchy.

class pyTooling.Warning.WarningCollector[source]

A context manager to collect warnings within the call hierarchy.

Inheritance

__init__(warnings=None, handler=None)[source]

Initializes a warning collector.

Parameters:

  • warnings (Optional[List[BaseException]]) – An optional reference to a list of warnings, which can be modified (appended) by this warning collector. If None, an internal list is created and can be referenced by the collector’s instance.

  • handler (Optional[Callable[[BaseException], bool]]) – An optional handler function, which processes the current warning and decides if a warning should be reraised as an exception.

Raises:

  • TypeError – If optional parameter ‘warnings’ is not of type list.

  • TypeError – If optional parameter ‘handler’ is not a callable.

Return type:

None
